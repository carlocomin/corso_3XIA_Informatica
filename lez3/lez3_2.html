L3.2 — IF/ELSE: esercizi svolti su varianti e pattern (testi completi)

Come useremo gli IF

  • Per ogni esercizio: Input, Output, Vincoli/Note, Esempi, quindi Soluzione a comparsa.
  • Varianti coperte: if, if–else, else–if ladder, if annidati con guardie.

E1 — Appartenenza a una unione di intervalli

Input: un intero x.
Output: la stringa "OK" se x ∈ [-10, -1] ∪ [10, 20], altrimenti "NO".
Vincoli/Note: intervalli chiusi agli estremi indicati; usare un’unica condizione logica.
Esempi: x=-10→OK; x=-1→OK; x=0→NO; x=10→OK; x=20→OK; x=21→NO. 
leggi x
cond ← ((x ≥ -10 ∧ x ≤ -1) ∨ (x ≥ 10 ∧ x ≤ 20))
se cond allora stampa "OK" altrimenti stampa "NO"

Frontiere: prova x = −10, −1, 10, 20.

E2 — Classificazione punto nel piano (assi, origine, quadranti)

Input: due reali x, y.
Output: una delle stringhe: "Origine", "Asse X", "Asse Y", "Q1", "Q2", "Q3", "Q4".
Vincoli/Note: gestire prima i casi speciali (origine, assi), poi i quadranti.
Esempi: (0,0)→Origine; (5,0)→Asse X; (0,-3)→Asse Y; (2,3)→Q1; (-1,4)→Q2; (-2,-2)→Q3; (3,-7)→Q4. 
leggi x, y
se (x = 0 ∧ y = 0) allora stampa "Origine"
altrimenti se (y = 0) allora stampa "Asse X"
altrimenti se (x = 0) allora stampa "Asse Y"
altrimenti se (x > 0 ∧ y > 0) allora stampa "Q1"
altrimenti se (x < 0 ∧ y > 0) allora stampa "Q2"
altrimenti se (x < 0 ∧ y < 0) allora stampa "Q3"
altrimenti stampa "Q4"

Sequenza corretta: prima casi speciali, poi quadranti.

E3 — Tariffa progressiva a scaglioni (costo totale)

Input: intero k (kWh, k ≥ 0).
Output: reale c, costo totale: primi 100 a 0.20, da 101 a 300 a 0.25, da 301 in poi a 0.30.
Vincoli/Note: rifiutare k < 0 con messaggio d’errore; somma per scaglioni.
Esempi: k=0→0.00; k=100→20.00; k=150→0.20·100+0.25·50=32.50; k=320→20+50+0.30·20=76.00. 
leggi k
se (k < 0) allora stampa "errore"
altrimenti
    c ← 0
    se (k ≤ 100) allora
        c ← 0.20 * k
    altrimenti se (k ≤ 300) allora
        c ← 0.20 * 100 + 0.25 * (k - 100)
    altrimenti
        c ← 0.20 * 100 + 0.25 * 200 + 0.30 * (k - 300)
    stampa c

Frontiere: k = 100, 101, 300, 301.

E4 — Giorni del mese con bisestile

Input: intero m (1..12), intero y (anno gregoriano).
Output: intero: giorni del mese; per febbraio: 29 se anno bisestile, altrimenti 28.
Vincoli/Note: bisestile se (y mod 400 = 0) ∨ ((y mod 4 = 0) ∧ (y mod 100 ≠ 0)).
Esempi: (2,2024)→29; (2,2100)→28; (4,2023)→30; (12,2022)→31; (13,2022)→errore. 
leggi m, y
se (m < 1 ∨ m > 12) allora stampa "errore"
altrimenti se (m = 1 ∨ m = 3 ∨ m = 5 ∨ m = 7 ∨ m = 8 ∨ m = 10 ∨ m = 12) allora stampa 31
altrimenti se (m = 4 ∨ m = 6 ∨ m = 9 ∨ m = 11) allora stampa 30
altrimenti
    bis ← (y mod 400 = 0) ∨ ((y mod 4 = 0) ∧ (y mod 100 ≠ 0))
    se (bis) allora stampa 29 altrimenti stampa 28

Gestione febbraio in ramo dedicato: chiaro e testabile.

E5 — Fasce voto (0..100) con ladder ordinata

Input: intero v (atteso 0..100).
Output: stringa: “insufficiente” (<60), “sufficiente” (60–69), “buono” (70–84), “ottimo” (85–94), “eccellente” (95–100), altrimenti errore.
Vincoli/Note: usare soglie crescenti per evitare sovrapposizioni.
Esempi: 59→insuff.; 60→suff.; 84→buono; 90→ottimo; 100→eccellente; -1→errore; 101→errore. 
leggi v
se (v < 0 ∨ v > 100) allora stampa "errore"
altrimenti se (v < 60) allora stampa "insufficiente"
altrimenti se (v < 70) allora stampa "sufficiente"
altrimenti se (v < 85) allora stampa "buono"
altrimenti se (v < 95) allora stampa "ottimo"
altrimenti stampa "eccellente"

Ladder in soglie crescenti ⇒ intervalli contigui, nessun buco.

E6 — Mediana di tre numeri senza ordinare l’intera terna

Input: tre numeri a, b, c (reali).
Output: la mediana (il valore centrale tra i tre).
Vincoli/Note: risolvere con if annidati, evitando un sort generale.
Esempi: (3,1,2)→2; (5,5,7)→5; (−2,10,0)→0. 
leggi a, b, c
se (a ≤ b) allora
    se (b ≤ c) allora med ← b
    altrimenti se (a ≤ c) allora med ← c
    altrimenti med ← a
altrimenti
    se (a ≤ c) allora med ← a
    altrimenti se (b ≤ c) allora med ← c
    altrimenti med ← b
stampa med

Massimo 4 confronti; copre tutte le permutazioni.

E7 — Conversione di temperatura con controllo unità

Input: valore reale t, carattere u in {'C','F'}.
Output: valore convertito e unità risultante (F se input in C, C se input in F); errore se u non valido.
Vincoli/Note: usare if–else con ramo di errore.
Esempi: (0,'C')→32 F; (212,'F')→100 C; (10,'X')→errore. 
leggi t, u
se (u = 'C') allora
    f ← t * 9/5 + 32
    stampa f, 'F'
altrimenti se (u = 'F') allora
    c ← (t - 32) * 5/9
    stampa c, 'C'
altrimenti
    stampa "errore unità"

Classico if–else-if con ramo “altrimenti” di errore.

E8 — Validazione di range con De Morgan

Input: interi a, b con a ≤ b, intero x.
Output: “ok” se x ∈ [a,b], “fuori range” altrimenti.
Vincoli/Note: evitare NOT esterno riscrivendo ¬(x<a ∨ x>b) con De Morgan.
Esempi: a=3, b=7: x=3→ok; x=7→ok; x=2→fuori; x=8→fuori. 
leggi a, b, x
se (x ≥ a ∧ x ≤ b) allora stampa "ok" altrimenti stampa "fuori range"

Uso esplicito di De Morgan: ¬(x<a ∨ x>b) ≡ (x≥a) ∧ (x≤b).

E9 — Multipli: scelta dal più specifico al generale

Input: intero n.
Output: “multiplo di 6” se n è multiplo di 6; altrimenti se multiplo di 3; altrimenti se multiplo di 2; altrimenti “nessuno”.
Vincoli/Note: ordinare i test per specificità per evitare rami errati.
Esempi: 12→“multiplo di 6”; 9→“multiplo di 3”; 8→“multiplo di 2”; 25→“nessuno”. 
leggi n
se (n mod 6 = 0) allora stampa "multiplo di 6"
altrimenti se (n mod 3 = 0) allora stampa "multiplo di 3"
altrimenti se (n mod 2 = 0) allora stampa "multiplo di 2"
altrimenti stampa "nessuno"

Prima 6 (più specifico), poi 3, poi 2.

E10 — Autorizzazione di accesso con guardie multiple

Input: booleano utenteAutenticato, stringa ruolo in {“admin”, “docente”, “studente”, …}, intero orario (0..23).
Output: “ACCESSO OK” se utenteAutenticato è vero e (ruolo ∈ {admin, docente} oppure orario ∈ [8,18]); altrimenti “NEGATO”.
Vincoli/Note: controllare prima l’autenticazione (guardia), poi i criteri di ruolo/orario.
Esempi: (true, “studente”, 9)→OK; (true, “admin”, 2)→OK; (false, “admin”, 10)→NEGATO; (true, “studente”, 20)→NEGATO. 
leggi utenteAut, ruolo, orario
perm ← utenteAut ∧ ( (ruolo = "admin" ∨ ruolo = "docente") ∨ (orario ≥ 8 ∧ orario ≤ 18) )
se perm allora stampa "ACCESSO OK" altrimenti stampa "NEGATO"

Frontiere: orario=8 e 18 inclusi; utente non autenticato nega sempre.

Checklist finale

  • Guardie in testa per evitare stati illegali (divisioni, range, unità).
  • Parentesi per chiarire la precedenza negli OR/AND.
  • Ladder ordinate per intervalli contigui; dal caso più specifico al più generale.
  • Frontiere sempre testate (estremi inclusi/esclusi).
