L3.1 — Decisioni & condizioni booleane (IF/ELSE) nei Flowchart

Obiettivi

  • Usare predicati e operatori relazionali e logici con precedenza corretta.
  • Disegnare pattern IF, IF–ELSE, ELSE–IF ladder e annidamento.
  • Semplificare condizioni con le leggi di De Morgan e progettare test di frontiera.

Logica di base: predicati, operatori, precedenza

  • Predicato: espressione booleana che vale vero o falso.
  • Relazionali: <, ≤, =, ≠, ≥, > (su numeri/char). Per stringhe: ordine lessicografico.
  • Logici: ¬ (NOT), ∧ (AND), ∨ (OR). Precedenza tipica: ¬ > ∧ > ∨. Usa parentesi per chiarezza.
AB¬AA ∧ BA ∨ B
FFVFF
FVVFV
VFFFV
VVFVV

De Morgan: ¬(A ∨ B) ≡ (¬A ∧ ¬B), ¬(A ∧ B) ≡ (¬A ∨ ¬B).

Pattern di Flowchart (spazi verticali ampi)

IF (solo se)
IF–ELSE
ELSE–IF ladder

Usa la ladder quando i casi sono mutuamente esclusivi e ordinati.

IF annidati (nidificazione)

Usa annidamento quando la scelta B dipende dal fatto che A sia vera.

Semplificare condizioni & short-circuit

  • In-range: a ≤ x ≤ b ⇔ (x ≥ a) ∧ (x ≤ b).
  • De Morgan: ¬(x∈A ∨ x∈B) ⇔ (x∉A) ∧ (x∉B) (evita doppie decisioni).
  • Short-circuit: in A ∧ B, se A è falso non si valuta B; in A ∨ B, se A è vero non si valuta B. Utile per guardie (es. b≠0 prima di a/b).

Esercizi (show/hide soluzioni)

  1. Appartenenza a intervallo chiuso — Dati x,a,b (con a≤b): stampa “OK” se x∈[a,b], altrimenti “NO”. 
    condizione: (x ≥ a) ∧ (x ≤ b)
IF condizione THEN stampa "OK" ELSE stampa "NO"
  2. Anno bisestile — Intero y: vero se (multiplo di 400) oppure (multiplo di 4 e non di 100). 
    cond: (y mod 400 = 0) ∨ ((y mod 4 = 0) ∧ (y mod 100 ≠ 0))
IF cond THEN stampa "BISESTILE" ELSE "NORMALE"
  3. Massimo di tre — Dati a,b,c, stampa il massimo (ladder). 
    IF (a ≥ b ∧ a ≥ c) THEN max ← a
ELSE IF (b ≥ a ∧ b ≥ c) THEN max ← b
ELSE max ← c
stampa max
  4. Divisione sicura — Leggi a,b; se b ≠ 0 stampa a/b, altrimenti messaggio di errore. 
    IF (b ≠ 0) THEN q ← a / b; stampa q
ELSE stampa "errore: divisione per zero"
  5. Classificazione voto — Intero v 0..10: stampa “insufficiente” (<6), “sufficiente” (6), “buono” (7–8), “ottimo” (9–10). 
    IF (v < 6)        THEN stampa "insufficiente"
ELSE IF (v = 6)   THEN stampa "sufficiente"
ELSE IF (v ≤ 8)   THEN stampa "buono"      // implica v=7 o 8
ELSE              stampa "ottimo"          // 9 o 10

Takeaways

  • Scrivi condizioni chiare con parentesi e nomi parlanti; verifica i casi di frontiera.
  • Preferisci la ladder per casi esclusivi e ordinati; usa annidamento quando il secondo test dipende dal primo.
  • Le De Morgan riducono i rami invertendo i test in modo meccanico.
