Strutture di controllo: if / else (C++ e C)

Obiettivi della lezione

  • Capire a cosa serve un if: prendere decisioni nel programma (“se succede X allora fai Y”).
  • Capire la differenza tra if semplice, if ... else e if ... else if ... else.
  • Capire come vengono valutate le condizioni booleane (true/false in C++, 1/0 in C).
  • Conoscere errori comuni: usare = al posto di ==, condizioni sempre vere, parentesi mancanti.
  • Vedere esempi completi con input da tastiera, output, e logica di decisione.

Idea chiave: il if è il punto in cui il programma “sceglie una strada”. Senza if il programma è solo una sequenza fissa di istruzioni sempre uguali.

Il concetto di condizione

Una condizione è un’espressione che può essere vera o falsa. Esempi di condizioni:

  • eta >= 18 (vero se l’utente ha almeno 18 anni)
  • nome == "Mario" (in C++ con std::string)
  • saldo < 0 (vero se stai “in rosso”)
  • temperatura > 37.5 (febbre?)

Il blocco if esegue un pezzo di codice solo se la condizione è vera.

In C++ la condizione è vera se l’espressione è true. In C la condizione è vera se l’espressione è diversa da 0 (cioè 1, 2, 100... sono considerati “vero”; 0 è “falso”).

Se una condizione non è chiarissima da leggere, mettila tra parentesi e magari spezzala in sotto-espressioni più semplici con variabili temporanee. Il codice deve essere leggibile, non “furbo”.

Struttura base: if semplice

Forma generale (C++ e C, stessa sintassi): 

if (condizione) {
    // blocco di istruzioni
}

Significa: se condizione è vera, esegui il blocco di istruzioni tra le { }. Se è falsa, salta quel blocco e continua dopo.

Esempio in C++

#include <iostream>
int main() {
    int eta;
    std::cout << "Quanti anni hai? ";
    std::cin >> eta;

    if (eta >= 18) {
        std::cout << "Sei maggiorenne.\n";
    }

    std::cout << "Fine del programma.\n";
    return 0;
}

Se l’utente inserisce ad esempio 20, la condizione eta >= 18 è vera ⇒ stampa “Sei maggiorenne.” Se inserisce 16, è falsa ⇒ non stampa quella riga.

Versione in C

#include <stdio.h>
int main(void) {
    int eta;
    printf("Quanti anni hai? ");
    scanf("%d", &eta);

    if (eta >= 18) {
        printf("Sei maggiorenne.\n");
    }

    printf("Fine del programma.\n");
    return 0;
}

Nota bene: stessa struttura logica, cambia solo lo stile I/O.

if ... else: scegliere tra due strade

Se vogliamo fare una cosa quando la condizione è vera e un’altra quando è falsa, usiamo if ... else. 

if (condizione) {
    // ramo 1: condizione vera
} else {
    // ramo 2: condizione falsa
}

Esempio in C++

#include <iostream>
int main() {
    int eta;
    std::cout << "Quanti anni hai? ";
    std::cin >> eta;

    if (eta >= 18) {
        std::cout << "Accesso consentito.\n";
    } else {
        std::cout << "Accesso negato.\n";
    }

    return 0;
}

Se la condizione (eta >= 18) è vera → primo blocco. Se è falsa → secondo blocco.

Versione equivalente in C

#include <stdio.h>
int main(void) {
    int eta;
    printf("Quanti anni hai? ");
    scanf("%d", &eta);

    if (eta >= 18) {
        printf("Accesso consentito.\n");
    } else {
        printf("Accesso negato.\n");
    }

    return 0;
}

Questo tipo di struttura è detto “se/altrimenti”: SE condizione allora fai A, ALTRIMENTI fai B. È la forma logica più comune nei programmi reali.

Catene di controllo: if ... else if ... else

Spesso non ci basta “vero vs falso”. Vogliamo più casi distinti (es. voto 0–5 = insufficiente, 6–7 = sufficiente, 8–9 = buono, 10 = eccellente). 

if (condizione1) {
    // caso 1
} else if (condizione2) {
    // caso 2
} else if (condizione3) {
    // caso 3
} else {
    // caso di default (nessuna condizione sopra era vera)
}

Esempio in C++

#include <iostream>
int main() {
    int voto;
    std::cout << "Inserisci voto (0-10): ";
    std::cin >> voto;

    if (voto < 6) {
        std::cout << "Insufficiente\n";
    } else if (voto <= 7) {
        std::cout << "Sufficiente\n";
    } else if (voto <= 9) {
        std::cout << "Buono\n";
    } else {
        std::cout << "Eccellente\n";
    }

    return 0;
}

Il flusso è: controlla la prima condizione. Se è vera, esegue quel blocco e poi salta tutto il resto. Se è falsa, prova la seconda, e così via.

Versione in C

#include <stdio.h>
int main(void) {
    int voto;
    printf("Inserisci voto (0-10): ");
    scanf("%d", &voto);

    if (voto < 6) {
        printf("Insufficiente\n");
    } else if (voto <= 7) {
        printf("Sufficiente\n");
    } else if (voto <= 9) {
        printf("Buono\n");
    } else {
        printf("Eccellente\n");
    }

    return 0;
}

Nota bene: appena una condizione è vera, le successive NON vengono valutate. Questo è importante per l’efficienza e per evitare di stampare più di una categoria.

Valori booleani (vero/falso)

  • In C++ il tipo bool ha solo due valori legali: true e false.
  • In C, tradizionalmente, if(condizione) considera “vero” qualunque cosa ≠ 0 e “falso” lo 0. (Da C99 esiste _Bool e <stdbool.h> che ti dà bool, true, false).
  • In entrambi i linguaggi vale che (a > b) è una condizione booleana: “è vero che a è maggiore di b?”

È buona abitudine usare condizioni che leggono bene ad alta voce: if (saldo < 0) si legge “se il saldo è minore di zero”. Evita cose criptiche tipo if (saldo) se non è chiaro cosa significa “saldo diverso da zero”.

Errori comuni (da evitare SUBITO)

  • Usare = invece di ==
    if (x = 5) { ... }
    Qui x = 5 è un’assegnazione, non un confronto. L’espressione vale 5, e 5 viene interpretato come “vero”. Quindi questo if è sempre vero e cambia anche il valore di x!
    if (x == 5) { ... }
    Ora stai facendo un confronto (“x è uguale a 5?”).
  • Dimenticare le parentesi graffe
    In C e C++ if (cond) istruzione; è legale senza { }. Ma appena aggiungi una seconda istruzione devi ricordarti di aggiungere { }. 
    if (x > 10)
    std::cout << "Grande\n";
    std::cout << "Sempre stampata\n"; // ATTENZIONE: NON fa parte dell'if!
    if (x > 10) {
    std::cout << "Grande\n";
    std::cout << "Solo se x>10\n";
}

    Per studenti alle prime armi: usa sempre le graffe, anche se c’è una sola istruzione.

  • Condizioni sempre vere o sempre false
    Se scrivi qualcosa tipo if (1) o if (true) non è un errore di sintassi, ma spesso è un errore logico: stai eseguendo sempre quel ramo.
  • Catene di else if incoerenti
    Se fai: 
    if (voto < 6) ...
else if (voto < 6) ...
    Il secondo ramo non verrà mai eseguito, perché è identico al primo. Controlla sempre gli intervalli logici (ad esempio < 6, poi <= 7, ecc.).

Esercizi (con soluzione a comparsa)

  1. E1 — Pari o dispari?
    Testo: Scrivi un programma che:
    • chiede un intero all’utente;
    • stampa “pari” se il numero è divisibile per 2, altrimenti “dispari”.
    Fai sia una versione C++ che una versione C.

    Soluzione C++:

    #include <iostream>
int main() {
    int n;
    std::cout << "Inserisci un numero intero: ";
    std::cin >> n;

    if (n % 2 == 0) {
        std::cout << "pari\n";
    } else {
        std::cout << "dispari\n";
    }

    return 0;
}

    Soluzione C:

    #include <stdio.h>
int main(void) {
    int n;
    printf("Inserisci un numero intero: ");
    scanf("%d", &n);

    if (n % 2 == 0) {
        printf("pari\n");
    } else {
        printf("dispari\n");
    }

    return 0;
}

    Qui la condizione è n % 2 == 0 (“il resto della divisione di n per 2 è zero?”). È un confronto, quindi usiamo ==, non =.

  2. E2 — Classificazione voti
    Testo: Scrivi un programma che legge un voto (0–10) e stampa:
    • “insufficiente” se < 6
    • “sufficiente” se 6 o 7
    • “buono” se 8 o 9
    • “eccellente” se 10
    Usa una catena if ... else if ... else. 
    #include <iostream>
int main() {
    int voto;
    std::cout << "Voto (0-10): ";
    std::cin >> voto;

    if (voto < 6) {
        std::cout << "insufficiente\n";
    } else if (voto <= 7) {
        std::cout << "sufficiente\n";
    } else if (voto <= 9) {
        std::cout << "buono\n";
    } else {
        std::cout << "eccellente\n";
    }
    return 0;
}

    Nota la logica sugli intervalli: if (voto < 6) prende 0..5, else if (voto <= 7) prende 6..7, else if (voto <= 9) prende 8..9, else prende solo 10.

  3. E3 — Controllo accesso sicuro
    Testo: Chiedi una password (stringa) all’utente e confrontala con quella “giusta”. Se è corretta, stampa “Accesso riuscito”, altrimenti “Accesso negato”.

    Spiegazione concettuale (C++):
    if (passwordInserita == passwordCorretta) funziona con std::string perché l’operatore == per std::string confronta il contenuto delle stringhe. 

    #include <iostream>
#include <string>
int main() {
    std::string pass;
    const std::string corretta = "segreto";

    std::cout << "Password: ";
    std::cin >> pass;

    if (pass == corretta) {
        std::cout << "Accesso riuscito\n";
    } else {
        std::cout << "Accesso negato\n";
    }
    return 0;
}

    In C classico il confronto tra stringhe non si fa con == ma con strcmp, perché le stringhe in C sono array di char. Questo è un anticipo di una difficoltà del C puro con le stringhe.

  4. E4 — Bug da scovare
    Testo: Trova l’errore logico qui sotto e spiega perché è pericoloso: 
    int x = 0;
if (x = 5) {
    std::cout << "OK\n";
} else {
    std::cout << "NO\n";
}

    Errore: x = 5 è un’assegnazione, non un confronto. L’assegnazione mette 5 in x e “vale” 5, che è considerato vero. Quindi il ramo else non verrà MAI eseguito. Andava scritto if (x == 5).

In sintesi
  • if decide se eseguire o no un blocco.
  • if ... else sceglie fra due blocchi.
  • if ... else if ... else modella più casi esclusivi (fasce di valori, categorie, stati diversi).
  • Usa sempre == per confrontare, NON =.
  • Chiudi SEMPRE i blocchi tra { }, anche se c’è una sola istruzione.
Prossima tappa (Lez6_8): quando abbiamo molti casi “a scelta multipla”, conviene usare switch.
