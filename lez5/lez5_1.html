Lez5_1 — Array in Flowgorithm: concetti, caricamento e stampa

Obiettivi didattici

  • Capire cos’è un array (collezione di celle omogenee, indicizzate).
  • Usare indici 0..n-1, con una capacità massima MAX e la dimensione logica n.
  • Caricare e stampare un array con cicli standard.
  • Leggere e scrivere pseudocodice chiaro e coerente con Flowgorithm.

Indici 0-based: primo elemento A[0], ultimo A[n-1]. Mai accedere a A[n].

Concetto di array (modello mentale)

  • Immagina una fila di cassette numerate 0..n-1, tutte dello stesso tipo (Integer/Real/String/Boolean).
  • Dichiariamo una capacità MAX (es. 100) e gestiamo la parte usata con n (≤ MAX).
  • Questo pattern semplifica validazioni, cicli e previene errori out-of-bounds.

Regola d’oro: ogni volta che usi A[i] deve valere 0 ≤ i < n.

Pseudocodice — Dichiarazione e caricamento

Idea: chiediamo n (1..MAX), poi leggiamo n valori riempiendo A[0..n-1].

Const MAX ← 100
Declare Integer A[MAX]     // array di interi con capacità MAX
Declare Integer n, i, x

// Richiedi e valida n
repeat
    Output "Inserisci n (1..", MAX, "): "
    Input n
until n ≥ 1 AND n ≤ MAX

// Caricamento dell'array
For i ← 0 To n-1 Step 1
    Output "A[", i, "] = ? "
    Input x
    A[i] ← x
End For

// Invariante (informale):
// Alla fine dell'iterazione con indice i sono corretti gli elementi A[0..i].

Attenzione: se n=0 l’array è vuoto → nessun accesso. In questa lezione imponiamo 1 ≤ n ≤ MAX.

Pseudocodice — Stampa sequenziale

Idea: percorriamo l’array in ordine e stampiamo ciascun elemento.

For i ← 0 To n-1 Step 1
    Output "A[", i, "] = ", A[i]
End For

// Complessità: O(n). Memoria extra: O(1).

Tabella di traccia (esempio di caricamento)

Dati: n = 5, sequenza inserita: [4, 7, 2, 9, 5].

4Input x=5; A[4]←5A = [4, 7, 2, 9, 5]

Pitfall comuni (e rimedi)

  • Indice fuori range (A[n], A[-1]).

    Usa sempre il limite To n-1 e valida n.
  • Dimenticare la dimensione logica.

    Stampa/usa solo A[0..n-1], non A[0..MAX-1].
  • n non valido.

    Ripeti richiesta finché 1 ≤ n ≤ MAX.

Esercizi (con soluzione a comparsa)

  1. Inverti stampa — Leggi n e A, poi stampa in ordine inverso.
    For i ← n-1 DownTo 0 Step 1
    Output A[i]
End For
  2. Conta positivi — Quanti elementi di A sono > 0?
    count ← 0
For i ← 0 To n-1 Step 1
    If A[i] > 0 Then
        count ← count + 1
    End If
End For
Output "Positivi = ", count

Takeaways

  • Array = collezione omogenea indicizzata 0..n-1.
  • Capacità MAX e dimensione logica n: accedi sempre entro i limiti.
  • Pattern fondamentali: caricamento e stampa con cicli.
  • Invariante: dopo l’iterazione i sono corretti A[0..i].
